# AI 视频生成 Prompt 指南

## 目标

FurTwin 后续会允许用户使用宠物照片生成绿幕宠物动作视频。生成的视频会被扣绿、序列帧化，并作为桌面宠物动作播放。

为了适合桌宠播放，AI 生成的视频需要满足：

- 纯绿幕背景；
- 固定机位；
- 宠物全身完整；
- 无文字、无水印、无 Logo；
- 写实风格；
- 动作幅度适中；
- 首尾可无缝循环；
- 方便 FFmpeg 扣绿处理。

## 推荐固定 Prompt 模板

```text
请严格按照图片中的动物，生成一只外形一模一样的。背景为纯绿幕背景，没有阴影、没有影子、没有光影变化，只有这个动物存在。全程保持它全身包括尾巴在视频中完整可见，动物主体始终完整位于画面中，四周保留少量安全边距，不要贴边。固定机位，不要拉近、不要推远、不要摇镜头。静音视频，不要声音。要写实风格，不要动漫风格。比例为16:9。不要出现任何文字、水印、Logo、字幕或标识。动作需要形成完整闭环，结尾姿态自然回到开头姿态，首尾可以无缝衔接循环播放，不要在结尾额外停顿。动作是：{用户输入的动作}。
```

## 示例 Prompt

```text
请严格按照图片中的动物，生成一只外形一模一样的。背景为纯绿幕背景，没有阴影、没有影子、没有光影变化，只有这个动物存在。全程保持它全身包括尾巴在视频中完整可见，动物主体始终完整位于画面中，四周保留少量安全边距，不要贴边。固定机位，不要拉近、不要推远、不要摇镜头。静音视频，不要声音。要写实风格，不要动漫风格。比例为16:9。不要出现任何文字、水印、Logo、字幕或标识。动作需要形成完整闭环，结尾姿态自然回到开头姿态，首尾可以无缝衔接循环播放，不要在结尾额外停顿。动作是：让它面向镜头舔爪子。
```

## 为什么要强调首尾循环

桌宠动画通常会循环播放。如果视频尾帧和首帧差异过大，转成序列帧后会出现明显跳变。因此 prompt 中需要强调：

- 动作形成完整闭环；
- 结尾自然回到开头姿态；
- 首尾可以无缝衔接；
- 不要在结尾额外停顿。

注意不要只写"首帧和尾帧完全相同"，因为这可能导致模型生成明显停顿。更推荐写"结尾姿态自然回到开头姿态，首尾可以无缝衔接循环播放"。

## 适合桌宠循环的动作

推荐动作：

- 舔爪子
- 摇尾巴
- 坐着眨眼
- 轻微转头
- 趴着呼吸
- 打盹
- 缓慢舔毛
- 轻微踩奶
- 趴着轻微摆动
- 原地小幅度伸懒腰

不推荐动作：

- 大幅跑动
- 跳跃
- 快速转身
- 离开画面
- 从站立变躺下但不回到起始姿态
- 包含多个复杂阶段的动作
- 和其他物体强交互的复杂动作

## 负面约束

生成视频时应尽量避免：

- 文字
- 水印
- Logo
- 字幕
- 其他动物
- 其他人或物体
- 地面阴影
- 绿幕光照变化
- 镜头推拉
- 镜头摇晃
- 宠物出画
- 尾巴或耳朵被裁切
- 动漫风格
- 拟人化动作

## 后续产品化建议

后续不要让用户完全自由输入 prompt。

推荐产品形态：

- 软件内置固定 prompt；
- 用户只输入动作描述；
- 提供推荐动作模板；
- 对复杂动作给出提示；
- 生成后先预览，用户确认后再进入扣绿和序列帧处理。
